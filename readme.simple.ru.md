# Контрфактические объяснения: Руководство для начинающих

## Что такое контрфактическое объяснение? (Простыми словами)

Представь, что ты подал заявку на кредит и получил отказ. Ты хочешь знать: "Что мне нужно изменить, чтобы одобрили?"

Именно это делает **контрфактическое объяснение**! Оно говорит: "Если бы твой доход был на 50 000 рублей выше, тебя бы одобрили."

> **Аналогия из жизни:** Представь видеоигру, где ты проиграл уровень. Вместо простого "Ты проиграл" игра говорит: "Если бы ты прыгнул на 2 секунды раньше, ты бы прошёл!" Это контрфакт — он объясняет, что нужно было изменить для другого результата.

## Как это применяется к трейдингу?

### Проблема с моделями-"чёрными ящиками"

Представь, что у тебя есть ИИ, который говорит, когда покупать или продавать Bitcoin. Он говорит:

```
Предсказание ИИ: ПРОДАВАТЬ Bitcoin!
Уверенность: 80%
```

Но ПОЧЕМУ? ИИ не объясняет себя. Это как друг, который даёт советы без объяснений:

```
Ты: "Мне покупать Bitcoin?"
Друг: "Продавай."
Ты: "Но почему?"
Друг: "Я просто знаю."
Ты: "???"
```

Не очень полезно!

### Контрфакты спешат на помощь!

Контрфактическое объяснение сказало бы:

```
Предсказание ИИ: ПРОДАВАТЬ Bitcoin!

НО...
Если бы RSI упал с 75 до 45, ИИ сказал бы ПОКУПАТЬ.
```

Теперь ты знаешь:
1. RSI (технический индикатор) важен для этого решения
2. Текущий RSI 75 (перекупленность) вызывает сигнал ПРОДАВАТЬ
3. Если рынок остынет (RSI до 45), ИИ переключится на ПОКУПАТЬ

> **Кулинарная аналогия:** Обычное объяснение: "Твой торт не получился из-за муки." Контрфакт: "Если бы ты использовал 2 стакана муки вместо 3, торт был бы идеальным!" Второй вариант точно говорит, что исправить!

## Понимание через примеры

### Пример 1: Решение про зонтик

**Обычный ИИ:** "Возьми сегодня зонтик."

**Контрфактический ИИ:** "Возьми сегодня зонтик. НО если бы вероятность дождя была 20% вместо 70%, зонтик не нужен."

| Параметр | Исходное | Контрфакт |
|----------|----------|-----------|
| Вероятность дождя | 70% | 20% |
| Температура | 18°C | 18°C |
| Ветер | Слабый | Слабый |
| **Решение** | Зонтик | Без зонтика |

**Что изменилось?** Только вероятность дождя! Это ключевой фактор.

### Пример 2: Крипто-трейдинг

**Исходная ситуация:**
```
RSI: 75 (перекупленность)
MACD: -0.5 (медвежий сигнал)
Объём: 1.2x от среднего
Тренд цены: Вниз
→ ИИ говорит: ПРОДАВАТЬ (80% уверенность)
```

**Контрфакт #1:**
```
RSI: 45 (изменено!)
MACD: -0.5
Объём: 1.2x от среднего
Тренд цены: Вниз
→ ИИ сказал бы: ДЕРЖАТЬ
```

**Контрфакт #2:**
```
RSI: 75
MACD: +0.3 (изменено!)
Объём: 1.2x от среднего
Тренд цены: Вверх (изменено!)
→ ИИ сказал бы: ПОКУПАТЬ
```

Теперь ты знаешь ДВА способа изменить решение!

## Почему это полезно?

### 1. Понимание риска

**Без контрфакта:**
"ИИ говорит ПОКУПАТЬ. Доверюсь ему."

**С контрфактом:**
"ИИ говорит ПОКУПАТЬ. Он переключится на ПРОДАВАТЬ, если RSI вырастет всего на 5 пунктов. Это рискованно — предсказание нестабильно!"

> **Аналогия:** Если мост выдерживает твою машину — хорошо. Но если ты знаешь, что он рухнет от лишнего килограмма, возможно, стоит поехать по другому мосту!

### 2. Поиск ключевых факторов

Контрфакты естественно показывают, что важнее всего:

```
Чтобы изменить ПРОДАВАТЬ → ПОКУПАТЬ:
Вариант A: Изменить RSI на -30 пунктов
Вариант B: Изменить MACD на +0.8
Вариант C: Изменить объём на -500%

Самое простое изменение = Самый влиятельный фактор
RSI легче изменить, значит это ключевой фактор!
```

### 3. Отладка стратегии

Иногда ИИ принимает странные решения. Контрфакты помогают понять почему:

```
Странное предсказание: ПРОДАВАТЬ когда всё выглядит бычьим?!

Контрфакт раскрывает:
"Если бы время изменилось с 3 ночи на 9 утра, предсказание = ПОКУПАТЬ"

Ага! ИИ научился никогда не торговать в 3 ночи.
Это баг или фича? Теперь ты можешь решить!
```

## Принцип "минимального изменения"

Хорошие контрфакты находят НАИМЕНЬШЕЕ нужное изменение:

```
ПЛОХОЙ контрфакт:
"Если бы RSI, MACD, объём, тренд и сентимент ВСЕ изменились,
 предсказание бы переключилось."
→ Бесполезно! Слишком много изменений.

ХОРОШИЙ контрфакт:
"Если бы ТОЛЬКО RSI изменился с 75 до 45,
 предсказание бы переключилось."
→ Полезно! Всего одно изменение.
```

> **Аналогия с навигатором:** Плохой навигатор говорит "Налево, потом направо, потом налево, потом разворот, потом направо", когда хороший говорит "Через 100 метров налево." Проще — лучше!

## Действенность: Что реально может измениться?

Не всё МОЖЕТ измениться. В трейдинге:

| Признак | Может измениться? | Почему? |
|---------|-------------------|---------|
| Вчерашняя цена | Нет | Прошлое фиксировано |
| Вчерашний объём | Нет | Прошлое фиксировано |
| Сегодняшний RSI | Да | Изменится с ценой |
| Сегодняшний MACD | Да | Изменится с ценой |
| Сентимент новостей | Нет | Внешний фактор |
| Время суток | Нет | Время идёт только вперёд |

Хороший контрфакт предлагает изменения только того, что МОЖЕТ реально измениться:

```
ПЛОХО: "Если бы вчерашняя цена была выше..."
→ Бесполезно! Прошлое не изменить.

ХОРОШО: "Если RSI упадёт на 10 пунктов..."
→ Полезно! RSI реально может измениться.
```

## Как работает генерация контрфактов

### Шаг 1: Начинаем с оригинала

```
Оригинал:
x = [RSI: 75, MACD: -0.5, Объём: 1.2]
Предсказание: ПРОДАВАТЬ
```

### Шаг 2: Делаем маленькие изменения

```
Пробуем: [RSI: 74, MACD: -0.5, Объём: 1.2] → Всё ещё ПРОДАВАТЬ
Пробуем: [RSI: 73, MACD: -0.5, Объём: 1.2] → Всё ещё ПРОДАВАТЬ
Пробуем: [RSI: 72, MACD: -0.5, Объём: 1.2] → Всё ещё ПРОДАВАТЬ
...
Пробуем: [RSI: 45, MACD: -0.5, Объём: 1.2] → ПОКУПАТЬ! Нашли!
```

### Шаг 3: Возвращаем объяснение

```
Контрфакт найден!
Измени RSI с 75 до 45 (уменьшить на 30)
Тогда предсказание станет: ПОКУПАТЬ
```

Компьютер делает это намного быстрее с помощью математики (оптимизации), но идея та же!

## Строительные блоки систем контрфактов

```
Твои торговые данные
     ↓
┌─────────────────────────┐
│ 1. ТОРГОВАЯ МОДЕЛЬ      │ ← Делает предсказания (ПОКУПАТЬ/ПРОДАВАТЬ)
│    ("Чёрный ящик")      │
└──────────┬──────────────┘
           ↓
┌─────────────────────────┐
│ 2. ГЕНЕРАТОР            │ ← Находит минимальные изменения
│    КОНТРФАКТОВ          │    для переключения предсказания
└──────────┬──────────────┘
           ↓
┌─────────────────────────┐
│ 3. ФОРМАТТЕР            │ ← Превращает в человеческие слова
│    ОБЪЯСНЕНИЙ           │
└──────────┬──────────────┘
           ↓
"Если RSI упадёт на 30, покупай вместо продажи"
```

## Аналогия из жизни: Игра "Измени оценку"

Представь, что получил тройку и хочешь пятёрку:

**Обычное объяснение:**
"Ты получил тройку, потому что домашка — 70%, тесты — 75%, участие — 80%."

**Контрфактическое объяснение:**
"Ты получил тройку. Чтобы получить пятёрку:
- Вариант 1: Поднять тесты с 75% до 95%
- Вариант 2: Поднять домашку с 70% до 90% И участие с 80% до 95%

Вариант 1 проще — сосредоточься на тестах!"

Это точно говорит, что изменить и какой вариант легче!

## Ограничения

### 1. Много валидных контрфактов

Иногда есть много способов переключить предсказание:

```
Чтобы изменить ПРОДАВАТЬ → ПОКУПАТЬ:
- Изменить RSI на -30 (с 75 до 45)
- ИЛИ изменить MACD на +0.8 (с -0.5 до +0.3)
- ИЛИ изменить и RSI на -15, и MACD на +0.4

Какой "лучший"? Зависит от того, что тебе важно!
```

### 2. Невозможные контрфакты

Иногда контрфакты невозможны:

```
"Если бы объём был отрицательным, предсказание бы переключилось"
→ Объём не может быть отрицательным! Плохой контрфакт.
```

Хорошие системы фильтруют такие случаи.

### 3. Это не то же самое, что причинность

```
Контрфакт: "Если бы RSI изменился, предсказание бы изменилось"
НЕ означает: "RSI ВЫЗЫВАЕТ движение рынка"

Корреляция ≠ Причинность
ИИ выучил паттерн; паттерн может не быть причинным.
```

## Попробуй сам: Упражнение на бумаге

### Материалы
- Бумага и карандаш
- Простое правило решения

### Твоё правило решения

```
ЕСЛИ (температура > 25°C) И (влажность > 60%)
ТО решение = "Остаться дома"
ИНАЧЕ решение = "Выйти на улицу"
```

### Упражнение 1: Найди контрфакты

Текущая ситуация:
- Температура: 28°C
- Влажность: 70%
- Решение: Остаться дома

**Вопрос:** Какое минимальное изменение для "Выйти на улицу"?

**Ответ:**
- Изменить температуру до 25°C или ниже (уменьшить на 3+), ИЛИ
- Изменить влажность до 60% или ниже (уменьшить на 10+)

Минимальное изменение = Температура до 25°C (всего 3 градуса!)

### Упражнение 2: Крипто-версия

Твоё простое торговое правило:
```
ЕСЛИ (RSI > 70) И (тренд цены = вниз)
ТО предсказание = ПРОДАВАТЬ
ИНАЧЕ ЕСЛИ (RSI < 30) И (тренд цены = вверх)
ТО предсказание = ПОКУПАТЬ
ИНАЧЕ предсказание = ДЕРЖАТЬ
```

Текущая ситуация:
- RSI: 75
- Тренд: Вниз
- Предсказание: ПРОДАВАТЬ

**Вопрос:** Какое изменение сделает ДЕРЖАТЬ?

**Ответ:**
- Изменить RSI с 75 до 70 или ниже, ИЛИ
- Изменить тренд с Вниз на Вверх или Нейтральный

## Ключевые выводы

1. **Контрфакты отвечают на "Что если?"** — Они объясняют, что должно измениться для другого результата.

2. **Проще — лучше** — Хорошие контрфакты меняют как можно меньше.

3. **Действенность важна** — Предлагай только изменения, которые реально могут произойти.

4. **Применение в трейдинге** — Понимай решения модели, управляй рисками, находи ключевые факторы.

5. **Существует много путей** — Часто есть больше одного способа переключить предсказание.

## Словарь

| Слово | Что означает | Пример |
|-------|--------------|--------|
| **Контрфакт** | Сценарий "что если" | "Если бы RSI был 45, я бы купил" |
| **Чёрный ящик** | Модель, внутрь которой не заглянуть | Нейронная сеть |
| **Близость** | Насколько близко к оригиналу | Изменение 1 признака vs 5 |
| **Валидность** | Переключает ли предсказание? | Да или Нет |
| **Действенность** | Может ли реально измениться? | RSI: Да, Прошлое: Нет |
| **Разреженность** | Количество изменённых признаков | 1 признак = очень разреженно |

## Интересные факты

1. **Термин пришёл из философии!** Философы веками обсуждали "контрфактические условные высказывания" (что-было-бы-если). Теперь ИИ использует ту же концепцию!

2. **Юридические системы используют контрфакты!** В праве тесты "если бы не" спрашивают: "Если бы не действие ответчика, произошёл бы вред?" Это контрфакт!

3. **Фильмы о путешествиях во времени — это контрфакты!** "Назад в будущее" по сути спрашивает: "Если Марти изменит прошлое, что случится с будущим?"

4. **Они помогают объяснять отказы!** GDPR Евросоюза фактически поощряет компании предоставлять контрфактические объяснения, когда ИИ отклоняет заявки на кредит.
